## 1. 产品定位 (重述)

**AI 深度参与的本地生活服务 + 招聘协作平台，以 API 为核心**。

不是给终端用户用的 App。是给"想搭一个 Boss 直聘 / 美团 / 闲鱼式 AI 协作平台"的前端团队或第三方系统，提供**所有能力的 API**。

5 类用户:
- 求职者 (job seeker)
- 招聘方 / BOSS / HR
- 服务提供者 (plumber / tutor / cleaner)
- 服务需求方 (homeowner)
- 平台管理员

10 个核心 API 模块 (用户原始 spec):

| # | 模块 | 关键能力 |
|---|------|----------|
| 1 | 用户与身份 | 多角色 + 实名认证 + AI 画像 |
| 2 | 信息发布 | 职位/简历/服务/商品 + AI 润色 |
| 3 | 智能匹配 | 人岗 + 服务-需求 + 直聊触达 |
| 4 | AI 即时通讯 | 托管助手 + 总结 + 日程 + 话术 |
| 5 | 审核风控 | 内容 + 虚假 + 反欺诈 |
| 6 | 交易流程 | 合同 + 报价 + 支付托管 |
| 7 | 评价信誉 | 真实性 + 多维分 |
| 8 | 数据洞察 | 行情 + 竞争力 + 运营大脑 |
| 9 | 协作任务 | 跨角色任务 + 智能通知 + AI 日志 |
| 10 | 开放集成 | Webhook + AI 模型插槽 + 数据导出 |

## 2. v0.2.0 现状 (基线)

6 层 + 1 横切:

```
┌─────────────────────────────────────────────────┐
│  api/         FastAPI 路由 (HTTP 入口)            │
│  protocol/    A2A / Skill Link 协议              │
│  collaboration/ 委托 / Task / Handoff            │
│  governance/  规则引擎 / 审计 / ACL             │
│  domain/      Candidate / JD / Match            │
│  identity/    AgentCard / Registry / Trust      │
│  infra/       持久化 / 事件 / Webhook            │
│  ─────────────────────────────────────────────── │
│  utils.py     Skill Link 格式 / state-error 翻译│
└─────────────────────────────────────────────────┘
```

**保留的 (30%)**:
- A2A Protocol — 直接对应 #10 开放集成
- AgentCard + TrustStrategy — 改造后对应 #1 + #7
- EventBus — 对应 #9 通知 + #10 Webhook
- AppendOnlyAuditLog (链式 hash) — 对应 #5 风控 + #9 AI 日志
- 6 层架构本身 — 工程底座
- Skill Link 响应格式 (utils.py) — API 形状

**改造的 (20%)**:
- `domain/Candidate/JD` → 扩成 `Resume/JobPost/Service/Product/Match`
- `identity/AgentCard` → 扩成多角色 Account
- `governance/engine.py` → 加 LLM 审核层
- `identity/auth.py` (APIKey) → 加手机号 / 微信 OAuth

**新建的 (70%)**: 见 §3

## 5. 数据流示例 (AI 协同创作)

```
[POST /api/posts] {title: "想找个会修水管的", category: "service", body: "..."}
  ↓
identity.AuthN 验证 token → account_id
  ↓
governance.content_audit (LLM) → risk_score, 关键词, 建议标签
  ↓
ai.enhance.enhance_text() → 润色标题 + 结构化字段
  ↓
domain.post.save(post) → event "post.created"
  ↓
infra.webhooks.dispatch() → 外部订阅者收到事件
  ↓
HTTP 201 {post_id, enhanced: {title, tags, fields}}
```
